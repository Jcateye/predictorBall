# PredictorBall UI 设计现状

> 设计文件：`designs/predictorBall.pen`
> 风格：Brutalist × Luxury 暗色移动端
> 屏幕宽度：402px（移动端标准）

---

## 1. 设计系统

### 1.1 色彩变量

| 变量名 | 色值 | 用途 |
|--------|------|------|
| `bg-primary` | `#0D0D0D` | 页面主背景 |
| `bg-card` | `#1A1A1A` | 卡片/区块背景 |
| `bg-muted` | `#2A2A2A` | 次级背景/筛选标签底色 |
| `accent-gold` | `#C9A962` | 主强调色（专家/付费/高亮） |
| `accent-blue` | `#42A5F5` | AI 标签/信息色 |
| `accent-green` | `#4CAF50` | 免费/成功/公开内容 |
| `accent-red` | `#E53935` | 警告/临场更新标签 |
| `live-red` | `#FF3B30` | 直播状态专用红 |
| `text-primary` | `#F5F4F2` | 主文字 |
| `text-secondary` | `#8A8A8A` | 次要文字 |
| `text-tertiary` | `#7A7A7A` | 三级文字 |
| `text-muted` | `#6A6A6A` | 弱化文字/辅助信息 |
| `text-disabled` | `#5A5A5A` | 禁用态文字 |
| `border-subtle` | `#2A2A2A` | 微弱边框 |
| `border-divider` | `#5A5A5A` | 分割线 |

### 1.2 字体规范

| 字体 | 用途 | 典型样式 |
|------|------|----------|
| **Bebas Neue** | 标题/标签/数字/Tab | 大写风格，letterSpacing 1~6，lineHeight 0.9 |
| **Cormorant Garamond** | 正文/摘要/描述 | italic 风格为主，fontSize 11~14 |
| **Inter** | 系统状态栏时间 | fontWeight 600，fontSize 17 |

### 1.3 设计规则

- **圆角**：0px（全局直角，Brutalist 风格）
- **阴影**：无（不使用任何 shadow）
- **图标**：Lucide 图标字体
- **间距**：页面内容区统一 padding `[0, 24]` 或 `[16, 24]`
- **底部 Tab Bar**：高度 84px，padding `[12, 0, 32, 0]`（含安全区）

## 2. 页面导航结构

| Tab | 图标 | 默认页面 | 说明 |

## 3. 各页面详细描述

### 3.1 Live 列表页

**节点 ID**：`LVa9e`
**页面层级**：Tab 一级页（实时 Tab 默认）

#### 结构

```
Status Bar
Header（"LIVE" 标题 + 直播场次计数指示器）
Filter Tabs（全部 / 热门 / 英超 / 西甲 / 意甲 / 我关注）
Match List
  ├── 联赛分组头（如"英超"）
  ├── 比赛卡片 × N
  │   ├── 左侧色条（红色=直播中，金色=即将开始）
  │   ├── 时间/状态（67' LIVE / VS 倒计时）
  │   ├── 主客队名 + 比分
  │   ├── 统计行（黄牌/角球数）
  │   └── 热门标记 / PRO 预测徽标
  └── "即将开始"分区
Tab Bar（实时 高亮）
```

#### 设计要点

- 比赛卡片左侧 4px 色条标识状态：`live-red` 表示直播中，`accent-gold` 表示即将开始
- 筛选标签可横向滚动，当前选中项用 `accent-gold` 底色
- HOT 标签使用 `accent-red` 背景
- PRO 预测徽标引导用户进入付费预测报告

#### 设计要点

- **全网分析卡片（Network Insight）**：平台级内容，默认置顶于卡片列表最上方
  - 使用 `accent-gold` 2px 描边边框，与普通卡片的 1px 边框区分
  - 头部 NETWORK INSIGHT 徽章：`globe` 图标 + 金色背景标签，强调平台身份
  - **增量信号格式**（核心差异化）：每条信号遵循四段结构：
    - 信号：发生了什么（事实陈述）
    - 影响：会影响什么 → 怎么影响（因果推导）
    - 置信度：●●○ 高/中/低 + 证据数量（金色实心圆 vs 描边空心圆）
    - 观察：什么时候/什么条件下验证（给用户下一步动作）
  - 信源透明度行：底部展示信源总数 + 分类（官方/记者/数据），建立可信度
  - CTA 按钮引导进入完整报告页，附更新时间戳
- **统一卡片模板**：专家/AI/网友三种作者类型共用同一卡片结构，通过颜色标签区分：
  - 金色 `accent-gold` → 专家
  - 蓝色 `accent-blue` → AI
  - 灰色 `text-muted` → 网友
- **二级筛选**（世界杯专属）：阶段、小组、时间、报告类型、价格、排序，均为下拉选择器
- 排序选项含"推荐/最新/热度/命中率"，命中率排序须满足最小样本门槛
- 卡片核心设计：不是"标题+价格"，而是**展示被锁住的价值**——先看到免费摘要，再看到锁定目录，最后 CTA
- 付费卡片用 `border-subtle` 边框，免费卡片用 `accent-green` 边框区分，全网分析卡片用 `accent-gold` 2px 边框

## 6. 作者体系

### 6.1 四种内容类型

| 类型 | 标签色 | 能否付费 | 说明 |
|------|--------|----------|------|
| 平台 | `accent-gold`（2px 描边） | 可以 | 全网分析（Network Insight），平台聚合多信源生成 |

| AI | `accent-blue` | 可以 | 平台 AI 生成报告 |
| 网友 | `text-muted` | 暂不开放 | UGC 内容，第一版仅免费发布 |

### 6.2 战绩展示规范（抗作弊）

- 命中率**永远带样本数**：显示 `71% (10/14)` 而非单独 `71%`
- 命中率排序须满足最小样本门槛（如近 30 天 ≥ 10 场）
- 推荐分 = 作者可信度（命中率 × 样本修正）+ 时效 + 行为 - 惩罚

### 6.3 网友发布规则（第一版）

- 默认仅免费发布
- 必须走统一模板（至少填：公开摘要、比赛、类型）
- 关键词风控：引导下注/拉群/私聊/代投/返利 → 拦截
- 付费仅开放给：认证专家 / 平台 AI / 官方数据报告

## 7. 全网分析报告内容结构

> 全网分析（Network Insight）是平台级内容产品，区别于专家/AI/网友的个人报告。
> 核心价值：用户一眼看到"最近发生了什么、会影响什么、怎么影响"。

### 7.1 免费区（卡片内直接展示）

#### 增量信号区（3~5 条）

每条信号遵循四段结构：

| 字段 | 说明 | 示例 |
|------|------|------|
| **信号** | 发生了什么（事实陈述） | 法国中卫瓦拉内训练缺席，出场存疑 |
| **影响** | 会影响什么 → 怎么影响（因果推导） | 防空能力下降 → 定位球风险显著上升 |
| **置信度** | ●●○ 高/中/低 + 证据数量 | ●●○ 中（2条证据） |
| **观察** | 什么时候/什么条件下验证 | 首发名单确认后验证，若缺阵关注替补质量 |

置信度等级：

| 等级 | 可视化 | 典型信源 |
|------|--------|----------|
| 高 | ●●● | 官方发布会、俱乐部公告、赛事组织方 |
| 中 | ●●○ | 跟队记者、训练场观察、多源交叉验证 |
| 低 | ●○○ | 天气预报、社交媒体传闻、单一信源 |

#### 信源透明度行

底部展示：`综合 N 条信源 · 官方 X / 记者 Y / 数据 Z`

信源分类：
- **官方**：赛事组织方、俱乐部/国家队官方发布
- **记者**：跟队记者、体育媒体记者
- **数据**：赔率数据、历史统计、体能/跑动数据

### 7.2 付费区（解锁后展示）

| 模块 | 内容 | 说明 |
|------|------|------|
| **比赛形态判断 + 置信度** | 对比赛走向的整体判断 | 如"高位逼抢消耗战，预计上半场节奏快、下半场体能分化" |
| **关键变量雷达** | 6 个子维度综合评估 | 阵容 / 体能 / 战术 / 定位球 / 天气 / 心理 |
| **情景推演** | 基于关键变量的 if-then 推演 | 如"A 首发 → 高位压迫成功率 70%"vs"A 缺阵 → 退守反击概率上升" |
| **风险点 + 反转条件** | 什么情况下判断会失效 | 如"若巴西提前进球，节奏控制权转移，低比分判断可能失效" |

#### 关键变量雷达（6 维度）

| 维度 | 评估内容 |
|------|----------|
| 阵容 | 首发/替补变化、伤停复出、阵型调整 |
| 体能 | 赛程密度、轮换程度、长途客场 |
| 战术 | 战术打法适配度、历史交锋战术演变 |
| 定位球 | 角球/任意球攻防数据、身高对比 |
| 天气 | 温度/湿度/降雨/风速对比赛的影响 |
| 心理 | 淘汰赛压力、连胜/连败心态、主场优势 |

### 7.3 与专家报告的区别

| 维度 | 全网分析（平台） | 专家报告（个人） |
|------|------------------|------------------|
| 信源 | 多信源聚合，标注来源分类 | 个人判断为主 |
| 免费区 | 增量信号（信号→影响→置信度→观察） | 一眼看懂理由（3条核心逻辑） |
| 付费区 | 形态判断 + 6维雷达 + 情景推演 + 风险点 | 比赛结论 + 关键变量 + 风险点 |
| 更新频率 | 持续追踪，多次更新 | 通常 1~2 次更新 |
| 卡片样式 | 金色 2px 描边 + NETWORK INSIGHT 徽章 | 标准 1px 边框 + 作者头像 |
